## Аутентификация

**Метод:** POST  

**URL:** `api/auth/login`  

**Описание:** аутентификация по логину (email) и паролю, access token, refresh
token и время жизни access токена в ответе

---

**Метод:** POST  

**URL:** `api/auth/refresh`  

**Описание:** обновление access токена, принимает refresh токен, возвращает access token, 
refresh token и время жизни access токена

---

**Метод:** GET  

**URL:** `api/auth/validate`  

**Описание:** метод для проверки валидности access токена

---

## Главная страница

**Метод:** GET  

**URL:** `api/projects?page={page}&size={size}&project-status={status}&project-type={type}&department={department}&title={title}&teamId={teamId}`

**Описание:** отображение краткой информации о проектах (название, кафедра, цель, тип проекта, 
статус) на главной странице с пагинацией и поиском по названию (кроме проектов со статусом
ON_VERIFICATION)  

**Параметры запроса**:
* `page` - число, необязательный (по умолчанию 1)
* `size` - число, необязательный (по умолчанию 9)
* `project-status` - строка, необязательный
* `project-type` - строка, необязательный
* `department` - строка, необязательный
* `title` - строка, необязательный
* `teamId` - число, необязательный

## Страница проекта

**Метод:** GET  

**URL:** `api/projects/{id}`  

**Описание:** Полная информация о проекте по id

---

**Метод:** PATCH  

**URL:** `api/projects/book/{id}`  

**Описание:** бронь проекта по его id, меняется статус проекта, добавляется новая
работа для команды (id пользователя берется из токена); доступен для авторизованных
пользователей с ролью STUDENT

---

**Метод:** PATCH  

**URL:** `api/projects/apply/{id}`  

**Описание:** принятие заявки на добавление проекта на витрину, меняется статус проекта; 
доступен для авторизованных пользователей с ролью ADMINISTRATOR и *TEACHER

---

**Метод:** PATCH  

**URL:** `api/projects/reject?projectId={projectId}&reason={reason}`  

**Описание:** отклонение заявки на добавление проекта на витрину, меняется статус проекта;
доступен для авторизованных пользователей с ролью ADMINISTRATOR и *TEACHER  

**Параметры запроса:**
* `projectId` - число, обязательный
* `reason` - строка, необязательный

**Описание:** получение полной информации о команде (id пользователя берется из токена);

**URL:** `api/teams/leave`  

**Описание:** исключить из команды пользователя с id из токена; доступен для авторизованных
пользователей с ролью STUDENT

**Описание:** исключить из команды пользователя по userId лидером команды, id которого

**URL:** `api/teams/leader/{userId}`  

**Описание:** изменить текущего лидера команды с id из токена на пользователя с userId;
доступен для авторизованных пользователей с ролью STUDENT
