# Витрина проектной деятельности

Бэкенд для веб приложения "Витрина проектной деятельности" (RestAPI)

## Быстрый старт

### Стек технологий:
- Java 22
- Spring Boot 4.0.3
- PostgreSQL 18.3
- Flyway 11.14.1
- Swagger (Open API) 3.0.2

### Требования для начала работы
- Java 22+
- Docker и Docker Desktop (для Postgres)
- Intellij IDEA / VS Code
- Git
- Postman (рекомендуется для запросов)
- DBeaver (рекомендуется для работы с бд)

### Клонирование репозитория

- Через Intellij IDEA (проще):
  - `Главный экран Intellij -> Clone Repository -> в поле URL вставить ссылку на репозиторий -> выбрать директорию -> Clone`
- Через терминал:
    - Перейти в папку для проектов: `cd D:\Java`
    - Клонировать репозиторий: `git clone ссылка на репозиторий showcase`
    - Перейти в папку проекта: `cd showcase`
- Ссылку можно получить на сайте гитхаба во вкладке Code

### Создание ветки

**Обязательно** сделать перед началом работы (от ветки `main`)

- Через Intellij IDEA:
  - `Кнопка master (правее кнопки с названием проекта) -> New Branch -> Указать имя ветки -> Create`
- Через терминал:
  - `git checkout -b anyclaw` — персональная ветка (вместо anyclaw свой никнейм)
- Рекомендуется в процессе разработки создавать другие ветки, отвечающие за конкретные фичи, например 
`git checkout -b anyclaw/auth`, но это не обязательно

### База данных

В Docker Desktop открыть терминал и вставить команду:
```
docker run --name projects_showcase -p 5432:5432 -e POSTGRES_USER=anyclaw -e POSTGRES_PASSWORD=1234 -d postgres
```
- --name: название контейнера
- -p: порт (первое число для публичного доступа, второе для доступа внутри контейнера)
- -e POSTGRES_USER: имя пользователя
- -e POSTGRES_PASSWORD: пароль
- -d postgres: версия Postgres (здесь поставит последнюю)

**Важно**: база данных на данном этапе должна быть пустой, без таблиц, заполняться она будет при помощи миграции Flyway
(об этом далее)

### Переменные окружения

- В корневой папке проекта создать файл `.env` (так и назвать)
- Скопировать содержимое файла `.env.example` в созданный ранее файл и вставить свои значения
- Левее кнопки с зелёным треугольником в выпадающем списка выбрать Edit Configuration
- Во вкладке Environment Variables выбрать ранее созданный файл, apply и ok

Для примера, переменные окружения для базы данных, созданной на предыдущем шаге:
```dotenv
DB_HOST=localhost
DB_PORT=5432
DB_DATABASE=postgres
DB_USER=anyclaw
DB_PASSWORD=1234
```

### Миграция Flyway

Миграции позволяют версионировать изменения структуры базы данных. Файлы миграций находятся в:
`src/main/resources/db/migration`. Там находятся файлы, каждый из которых отвечает за конкретную
версию бд с чёткой структурой:

`V1__Init_DB.sql`
- V - обязательно
- 1 - номер версии (меняется вместе с версией)
- __ - два нижних подчёркивания (обязательно)
- Init_DB - краткое описание версии (меняется вместе с версией)
- 
**Важно:** никогда не изменяйте уже существующие файлы миграций. Только создавать новые!
Для создания новой версии:
1. Создать файл `V2__add_new_column.sql`
2. Написать SQL-запросы (ALTER TABLE, CREATE TABLE и т.д.)
3. Запустить приложение — миграция применится автоматически

### Commit, push и pull request

После внесений изменений в проект (реализации части фичи и тп.) надо сделать коммит в свою ветку:
- Через Intellij IDEA:
  - `Кнопка с названием ветки, правее названия проекта -> Commit -> выбрать файлы для коммита -> описание изменений через запятую 
-> commit`
- Через терминал:
  - `git add .`, точка означает включить все файлы, если надо конкретные файлы, тогда их перечислить
  - `git commit -m "Add user profile endpoint"`, в кавычках описать изменения через запятую

Когда какая-нибудь задача будет готова, следует сделать push (его можно сделать сразу после коммита):
- Через Intellij IDEA:
    - `Кнопка с названием ветки, правее названия проекта -> Push -> кнопка Push`
- Через терминал:
    - `git push origin branchname`, вместо branchname указать название своей ветки

Изменения внесутся только в конкретную ветку, чтобы внести изменения в главную ветку нужно сделать Pull request:
- На сайте гитхаба появится кнопка Create pull request, нажать
- Если не появилась, `Вкладка Pull Requests -> New pull request`
- Выбрать compare ветку свою, base ветку main
- Выбрать коммит, описать pull request и отправить
- Другие пользователи смогут комментировать код с указанием на конкретные его части
- После одобрения ревьювера кнопку `Merge pull request` нажимает **автор PR**

### Документация SWAGGER

Swagger генерирует html страницу с информацией о всех эндпоинтах (метод, принимаемые и возвращаемые
значения, возможные коды). Чтобы открыть Swagger документацию:

- Запустить проект
- Перейти по адресу: `http://localhost:8080/swagger-ui.html`, если порт 8080
- Чтобы открыть документацию в формате JSON: `http://localhost:8080/v3/api-docs`
